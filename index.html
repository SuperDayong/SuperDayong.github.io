SSH服务安装与无密码登陆设置

安装SSH服务

SSH简介：

  • SSH 为 Secure Shell 的缩写，由 IETF 的网络工作小组（Network Working Group）所制定；
  • SSH 为建立在应用层和传输层基础上的安全协议。SSH 是目前较可靠，专为远程登录会话和其他网络服务提供安全性的协议。
  • 利用 SSH 协议可以有效防止远程管理过程中的信息泄露问题。
  • 透过 SSH 可以对所有传输的数据进行加密，也能够防止 DNS 欺骗和 IP 欺骗。

SSH安装

  • 这里我用的是CentOS7,OpenSSH是默认安装的。
  • 检测SSH是否安装？ 打开终端，输入ssh，如下所示则表明已经安装了SSH
[root@localhost ~]# ssh
  • usage: ssh [-1246AaCfgKkMNnqsTtVvXxYy] [-b bind_address] [-c cipher_spec]
  • [-D [bind_address:]port] [-E log_file] [-e escape_char]
  • [-F configfile] [-I pkcs11] [-i identity_file]
  • [-L [bind_address:]port:host:hostport] [-l login_name] [-m mac_spec]
  • [-O ctl_cmd] [-o option] [-p port]
  • [-Q cipher | cipher-auth | mac | kex | key]
  • [-R [bind_address:]port:host:hostport] [-S ctl_path] [-W host:port]
  • [-w local_tun[:remote_tun]] [user@]hostname [command]

若没有安装，则在终端执行下面的命令进行安装

yum install ssh

SSH启动

[root@localhost ~]# service sshd start

SSH开机运行

[root@localhost ~]# chkconfig sshd on

检查配置

Hadoop需要通过SSH来启动Slave节点的守护进程，即使安装伪分布式也需要SSH。
除了ssh外还需要rsync服务，rsync是一个远程数据同步工具，可通过LAN/WAN快速同步多台主机间的文件。
判断ssh和rsync有没有安装

rpm –qa | grep openssh

rpm –qa | grep rsync

如果没有安装使用以下命令安装

yum install ssh 安装SSH协议

yum install rsync

配置免密码登录

免密码登录的原理

  • Master（NameNode | JobTracker）作为客户端，要实现无密码公钥认证，连接到服务器Salve（DataNode | Tasktracker）上时，
    需要在Master上生成一个密钥对，包括一个公钥和一个私钥，而后将公钥复制到所有的Slave上。

  • 当Master通过SSH连接Salve时，Salve就会生成一个随机数并用Master的公钥对随机数进行加密，并发送给Master。

  • Master收到加密数之后再用私钥解密，并将解密数回传给Slave，Slave确认解密数无误之后就允许Master进行连接了。

这就是一个公钥认证过程，其间不需要用户手工输入密码。
重要过程是将客户端Master复制到Slave上。

免密码登录实现

1，使用普通用户登录CentOS。进入用户的home目录，执行如下命令

$ ssh-keygen -t rsa -P '' -f ~/.ssh/id_rsa

注解：

  1. ssh-keygen表示生成密钥；
  2. -t指定密钥类型；
  3. -P 提供密语；
  4. -f生成的密钥文件

这条语句表示生成了一个无密码密钥对，此时目录下会生成id_rsa和id_rsa.pub两个文件，前者是私钥，后者是公钥

2，将公钥追加到授权key中

$ cat ~/.ssh/id_rsa.pub >> ~/.ssh/authorized_keys

3，将授权key文件权限改为600（rr——-）

$ chmod 600 ~/.ssh/authorized_keys

4,切换到root用户，修改ssh配置文件

vim /etc/ssh/sshd_config

修改其中部分内容（去掉原来的注释符号）

  1. RSAAuthentication yes # 启用 RSA 认证
  2. PubkeyAuthentication yes # 启用公钥私钥配对认证方式
  3. AuthorizedKeysFile .ssh/authorized_keys # 公钥文件路径（和上面生成的文件同）

退出root用户，验证是否可以免密码登陆

[root@localhost ~]# ssh localhost

Last login: Fri Apr 15 12:04:33 2016 from localhost

这样就成功的登陆SSH了

CentOS7 JDK 环境搭建

发表于 2016-04-11   |   分类于 Linux   |  

主要准备工具

  1. VMware（这里我用的是12版）
  2. CentOS-7-x86_64-DVD-1511.iso（有需要的可以找我要网盘地址）

JDK操作步骤

检查JDK版本

[dayong@localhost ~]$ java -version
  • java version “1.7.0_51”
  • OpenJDK Runtime Environment (rhel-2.4.5.5.el7-x86_64 u51-b31)
  • OpenJDK 64-Bit Server VM (build 24.51-b03, mixed mode)

检查JDK安装包

[dayong@localhost ~]$ rpm -qa | grep java
  • python-javapackages-3.4.1-5.el7.noarch
  • javapackages-tools-3.4.1-5.el7.noarch
  • java-1.7.0-openjdk-headless-1.7.0.51-2.4.5.5.el7.x86_64
  • tzdata-java-2014b-1.el7.noarch
  • java-1.7.0-openjdk-1.7.0.51-2.4.5.5.el7.x86_64

卸载CentOS自带的OpenJDK（不建议使用自带的OpenJDK）

[dayong@localhost ~]$ rpm -e --nodeps tzdata-java-2014b-1.el7.noarch
  • 错误：can’t create 事务 lock on /var/lib/rpm/.rpm.lock (权限不够)
  • 这一步就是说 dayong 账号没权限，很简单，我们直接用root账号（当然也可以赋权限，先简单着来吧）
[dayong@localhost ~]$ su root 
[root@localhost ~]# rpm -e --nodeps tzdata-java-2014b-1.el7.noarch
[root@localhost ~]# rpm -e --nodeps java-1.7.0-openjdk-headless-1.7.0.51-2.4.5.5.el7.x86_64
[root@localhost ~]# rpm -e --nodeps java-1.7.0-openjdk-1.7.0.51-2.4.5.5.el7.x86_64

这样原有的JDK就被我们卸载干净了，接下来我们就可以安装自己下载的JDK了

安装JDK

首先进入到JDK压缩包的路径下

cd /home/dayong/下载
  • [root@localhost Downloads]# ls -l
  • 总用量 149936
  • -rw-rw-r–. 1 dayong dayong 153530841 4月 11 13:14 jdk1.7.0_79.tar.gz

将文件移到新建的java文件夹下面

cp jdk1.7.0_79.tar.gz /usr/local/java

到java文件夹路径下解压压缩包，并删除原压缩包

tar xvf jdk1.7.0_79.tar.gz
rm jdk1.7.0_79.tar.gz

配置JDK环境变量

这里直接采用全局设置，修改文件 etc/profile 文件

gedit /etc/profile 或者 vi /etc/profile

在文件末尾添加如下几行环境变量配置:

export JAVA_HOME=/usr/local/java/jdk1.7.0_79
export JRE_HOME=/usr/local/java/jdk1.7.0_79/jre
export CLASSPATH=.:$JAVA_HOME/jre/lib/rt.jar:$JAVA_HOME/lib/dt.jar:$JAVA_HOME/lib/tools.jar
export PATH=$PATH:$JAVA_HOME/bin

保存并退出文件，接下来使文件生效

1，source /etc/profile   --使文件生效
2，reboot --重启生效

在终端输入

java -version

到这里，JDK环境已经搭配好了

第一篇Hexo博客

更换主题

1，首先在Git上寻找自己需要的主题，在博客目录 D:\hexo 下右键点击Git Bash，输入以下命令

git clone https://github.com/iissnan/hexo-theme-next themes/next

这样就完成了第一步，并且会出现 D:\HEXO\themes\next 这个文件夹

2，打开 D:\HEXO 目录下的 (下划线)config.yml 文件，将 theme 改为 next

保存并退出即可

3,本地测试

依次执行下面语句

hexo g #生成静态网页 
hexo s #开启预览访问端口（默认端口4000，'ctrl + c'关闭server）

打开浏览器，输入： http://localhost:4000/

–这样本地的博客就建好了，怎么样，很简单吧！

4，下一步我们要做的就是部署

hexo deploy  #进行部署

然后可以使用Github用户名.github.io进行访问, 或者设置个性域名，最晚10分钟左右，即可看到自己的效果
