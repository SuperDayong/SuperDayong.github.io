CentOS7 JDK 环境搭建

发表于 2016-04-11   |   分类于 Linux   |  

主要准备工具

  1. VMware（这里我用的是12版）
  2. CentOS-7-x86_64-DVD-1511.iso（有需要的可以找我要网盘地址）

JDK操作步骤

检查JDK版本

[dayong@localhost ~]$ java -version
  • java version “1.7.0_51”
  • OpenJDK Runtime Environment (rhel-2.4.5.5.el7-x86_64 u51-b31)
  • OpenJDK 64-Bit Server VM (build 24.51-b03, mixed mode)

检查JDK安装包

[dayong@localhost ~]$ rpm -qa | grep java
  • python-javapackages-3.4.1-5.el7.noarch
  • javapackages-tools-3.4.1-5.el7.noarch
  • java-1.7.0-openjdk-headless-1.7.0.51-2.4.5.5.el7.x86_64
  • tzdata-java-2014b-1.el7.noarch
  • java-1.7.0-openjdk-1.7.0.51-2.4.5.5.el7.x86_64

卸载CentOS自带的OpenJDK（不建议使用自带的OpenJDK）

[dayong@localhost ~]$ rpm -e --nodeps tzdata-java-2014b-1.el7.noarch
  • 错误：can’t create 事务 lock on /var/lib/rpm/.rpm.lock (权限不够)
  • 这一步就是说 dayong 账号没权限，很简单，我们直接用root账号（当然也可以赋权限，先简单着来吧）
[dayong@localhost ~]$ su root 
[root@localhost ~]# rpm -e --nodeps tzdata-java-2014b-1.el7.noarch
[root@localhost ~]# rpm -e --nodeps java-1.7.0-openjdk-headless-1.7.0.51-2.4.5.5.el7.x86_64
[root@localhost ~]# rpm -e --nodeps java-1.7.0-openjdk-1.7.0.51-2.4.5.5.el7.x86_64

这样原有的JDK就被我们卸载干净了，接下来我们就可以安装自己下载的JDK了

安装JDK

首先进入到JDK压缩包的路径下

cd /home/dayong/下载
  • [root@localhost Downloads]# ls -l
  • 总用量 149936
  • -rw-rw-r–. 1 dayong dayong 153530841 4月 11 13:14 jdk1.7.0_79.tar.gz

将文件移到新建的java文件夹下面

cp jdk1.7.0_79.tar.gz /usr/local/java

到java文件夹路径下解压压缩包，并删除原压缩包

tar xvf jdk1.7.0_79.tar.gz
rm jdk1.7.0_79.tar.gz

配置JDK环境变量

这里直接采用全局设置，修改文件 etc/profile 文件

gedit /etc/profile 或者 vi /etc/profile

在文件末尾添加如下几行环境变量配置:

export JAVA_HOME=/usr/local/java/jdk1.7.0_79
export JRE_HOME=/usr/local/java/jdk1.7.0_79/jre
export CLASSPATH=.:$JAVA_HOME/jre/lib/rt.jar:$JAVA_HOME/lib/dt.jar:$JAVA_HOME/lib/tools.jar
export PATH=$PATH:$JAVA_HOME/bin

保存并退出文件，接下来使文件生效

1，source /etc/profile   --使文件生效
2，reboot --重启生效

在终端输入

java -version

到这里，JDK环境已经搭配好了
